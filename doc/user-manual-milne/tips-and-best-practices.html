In Which We Share Some Useful Things

Here are some things that people have discovered while using Dydra Studio. They're the sort of things you learn by doing, but we thought we'd write them down so you don't have to discover them all by yourself.

Getting Around

  • The Location Bar: There's a bar at the top where you can type addresses directly. It's faster than clicking, sometimes.
  • Back and Forward: The browser's back and forward buttons work just like they should. Very handy.
  • Tabs: You can open several things at once in different tabs. The computer doesn't mind keeping track.
  • Shortcuts: Some things have keyboard shortcuts:
    • Ctrl+Enter (or Cmd+Enter on a Mac): Run a query
    • Enter: Submit a form
    • Escape: Close a dialog (when you change your mind)

The Query Editor

  • Colours: The editor colours different parts of your SPARQL. Pay attention to the colours‚Äîthey help you see if something's wrong.
  • Suggestions: Sometimes the editor suggests words while you type. It's trying to help.
  • Multiple Tabs: You can have several queries open at once and compare them.
  • Different Formats: Try different answer formats to see which one suits your needs.
  • The Log: If something goes wrong, check the log. It remembers everything that happened.

Looking After Your Data

  • Make Copies: Before you do anything big, export your data first. Then if something goes wrong, you haven't lost everything.
  • Start Small: When importing, try a little bit first to make sure it works before importing the whole lot.
  • Let Big Things Run: Large imports can work in the background. You don't have to watch them the whole time.
  • Watch the Progress: Those little progress indicators tell you how things are going. They're worth watching.
  • Check Afterwards: After importing, run a quick query to make sure everything arrived properly.

Sharing Nicely

  • Start Carefully: When you add new people, give them read-only access first. You can always give them more later.
  • Write It Down: Remember why you gave each person access. You'll forget otherwise.
  • Check Sometimes: Every now and then, look at who has access and remove anyone who doesn't need it anymore.
  • Talk to People: Let your collaborators know what you expect. It prevents misunderstandings.

Keeping Things Running Smoothly

  • Don't Open Too Much: Having lots of tabs open makes everything slower. Close the ones you're not using.
  • Use Search: Instead of scrolling through everything, use filters to find what you need.
  • Write Good Questions: Well-written SPARQL runs faster. It's worth learning to write it well.

When Things Go Wrong

Import Troubles

If your data won't go in:

  • Is it a file type that's supported?
  • Is the file too big?
  • Do you have permission to write here?
  • Try using the background import for big files

Query Troubles

If your question won't work:

  • Is there a spelling mistake in your SPARQL?
  • Is there data there to ask about?
  • What does the error message say?
  • Try a simpler question first

Login Troubles

If you can't get in:

  • Is your password right? (Check the Caps Lock key.)
  • Is the host address correct?
  • Try clearing your browser's memory and starting fresh
  • Ask the administrator if nothing works

Collaboration Troubles

If sharing isn't working:

  • Are the usernames spelled correctly?
  • Are you allowed to manage collaborators?
  • Did you click Save?
  • Try refreshing the page

The Most Important Hint of All

Click Save.

Whatever you do, remember to click Save. The computer can't read your mind. It only knows about changes when you tell it to save them. So always, always click Save.

(We've said this before, but it's worth saying again.)
