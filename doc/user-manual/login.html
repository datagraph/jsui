Login and Authentication

Overview

Dydra Studio requires authentication to access your account and repositories. This section explains how to log in to the application and manage your session.

Accessing the Login Page

To access Dydra Studio, navigate to:

URL: https://dydra.com/ui/user

The application will display the login page when you first access it or when you are not authenticated.

Logging In

To log in to your Dydra account:

Step 1: Navigate to the login page at https://dydra.com/ui/user

Figure 1: Login form with credentials entered, ready to submit

Step 2: Enter your account credentials:
  • Host: The Dydra service host (typically https://dydra.com)
  • Login: Your account username
  • Password: Your account password
Step 3: Click the Login button or press Enter
Step 4: Upon successful authentication, you will be redirected to your account page, where you can see your repositories and account information.
Note: The application supports multiple authenticated accounts simultaneously. You can switch between accounts or add additional accounts during your session.

Authentication Methods

Dydra Studio supports two authentication methods:

  • Basic Authentication: Username and password authentication
  • Bearer Token Authentication: Token-based authentication (for programmatic access)
    Express this as "Bearer tokenValue"

For normal user access, use Basic Authentication with your username and password.

Session Management

Session Persistence

The application persists only account name(s) across page reloads using browser storage. This means:

  • Account names you have logged into are remembered when you reload the page
  • You can quickly access previously used accounts without re-entering the account name
Important: Authentication tokens and all other session data exist only in JavaScript objects during a single page session. When you close the page or browser tab, all session data (including tokens) is discarded. You will need to log in again after closing the page, even if account names are remembered.

Logging Out

To log out of the application:

  1. Click the Logout link in the navigation bar
  2. You will be redirected to the login page
  3. All session data (including authentication tokens) will be cleared from memory
  4. Account names may still be remembered in browser storage for convenience, but you will need to log in again to access your account

Troubleshooting

Login Failures

If you are unable to log in, check the following:

  • Verify that your username and password are correct
  • Ensure that the host URL is correct (typically https://dydra.com)
  • Check your internet connection
  • Verify that your account exists and is active

Error Messages

If you see an error message during login:

  • Read the error message carefully - it will indicate the specific problem
  • Common errors include "Invalid credentials" or "Authentication failed"
  • If the problem persists, contact support at support@dydra.com
