Overview

This section covers all operations related to managing your Dydra account, including viewing account information, editing account metadata, and managing authentication tokens.

Viewing Account Information

After logging in, you will be taken to your account page, which displays:

  • Account friendly ID (username)
  • Email address
  • Full name
  • Homepage URL
  • Blog URL
  • Company name
  • Location
  • List of repositories

Editing Account Metadata

You can edit various account profile fields to keep your account information up to date.

Editable Fields

The following account fields can be edited:

Field Description
firstname Your first name
familyname Your family/last name
fullname Your full name
email Your email address
homepage URL to your personal homepage
blog URL to your blog
company Your company name
location Your location
phone Your phone number
skype_id Your Skype ID
jabber_id Your Jabber/XMPP ID
workinfo Work-related information

How to Edit Account Metadata

Step 1: Navigate to your account page by clicking on your account name in the navigation or by accessing the account directly.

Figure 1: Account metadata field in edit mode with new value entered

Step 2: Locate the field you want to edit in the account information section.
Step 3: Double-click on the field value to enter edit mode. The field will become an editable input field.
Step 4: Enter or modify the value in the input field.
Step 5: Press Enter or click outside the field to save the change. The field will return to display mode with your updated value.
Step 6: Click the Save button (usually located in the tab bar) to persist your changes to the server.

Figure 2: Account page showing the Save button in the tab bar

Note: The application tracks only the fields you have edited. When you click Save, only the modified fields are sent to the server, not the entire account configuration.
Important: Changes are not saved to the server until you click the Save button. If you navigate away from the page without saving, your changes will be lost.

Viewing Authentication Tokens

Your account page displays your authentication token, which can be used for programmatic access to the Dydra API.

Accessing Your Token

  1. Navigate to your account page
  2. Look for the "Token" section in the sidebar or account information area
  3. Your current authentication token will be displayed

Resetting Your Token

If you need to generate a new authentication token:

  1. Navigate to your account page
  2. Find the token section
  3. Click the Reset Token button (if available)
  4. A new token will be generated and displayed
  5. Update any applications or scripts that use the old token

Figure 3: Account page showing token section with Reset Token button

Security Warning: Keep your authentication token secure. Anyone with access to your token can access your account. If you suspect your token has been compromised, reset it immediately.

Account Navigation

From your account page, you can:

  • View all your repositories
  • Create new repositories
  • Navigate to specific repositories
  • Access repository management features

Multiple Accounts

Dydra Studio supports managing multiple authenticated accounts simultaneously. This allows you to:

  • Switch between different accounts
  • Work with repositories from multiple accounts
  • Manage accounts with different permissions
Note: Each account maintains its own session and authentication state. You can have multiple account tabs open simultaneously.

Troubleshooting

Changes Not Saving

If your account metadata changes are not being saved:

  • Ensure you clicked the Save button after making changes
  • Check for error messages that might indicate a problem
  • Verify that you have permission to edit the account
  • Check your internet connection

Token Not Displaying

If your authentication token is not visible:

  • Ensure you are logged in to the correct account
  • Check that you have permission to view tokens
  • Try refreshing the page
