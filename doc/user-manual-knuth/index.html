Preface

This manual describes Dydra Studio, a browser-based application for managing RDF (Resource Description Framework) data repositories. The name "Dydra" derives from the computational notion of persistent, distributed data structures—though the etymological details need not concern the practitioner.

The Resource Description Framework, standardized by the World Wide Web Consortium in 1999 and subsequently revised, provides a formal model for expressing relationships between entities. Each statement in RDF constitutes a triple: a subject, predicate, and object. When extended with a fourth component specifying the named graph, we obtain a quad. This manual assumes familiarity with these fundamental concepts.

Scope and Purpose

Dydra Studio is implemented as what contemporary web developers term a "single-page application"—a design pattern wherein the interface updates dynamically without requiring full page reloads. This architectural choice has important implications for both performance and user interaction.

The system provides facilities for:

  • Persistent storage of RDF datasets in named repositories
  • Query execution using the SPARQL Protocol and RDF Query Language
  • Data interchange via multiple serialization formats
  • Management of saved queries (termed "views")
  • Collaborative access control mechanisms
  • Modification of metadata associated with repositories and accounts

Prerequisites

System Requirements

The application requires a modern web browser with JavaScript execution capabilities. Empirical testing confirms compatibility with recent versions of Chrome, Firefox, Safari, and Edge. The reader is advised to maintain current browser versions, as older implementations may lack necessary API support.

Technical Note: The application URL is https://dydra.com/ui/user. This constitutes the entry point for all subsequent operations described herein.

Interface Variants

Two distinct interfaces exist:

  • User Application: The primary interface for data management operations, documented in this manual.
  • Administrative Console: A separate interface for system administration, documented elsewhere.

Algorithm: Initial Access

The procedure for initial system access may be expressed as follows:

  1. Direct the browser to https://dydra.com/ui/user
  2. Complete the authentication procedure (see Chapter 1)
  3. Proceed with desired operations as documented in subsequent chapters
