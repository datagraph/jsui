Questions

Views are questions. You save them so you don't have to type them again. The computer remembers them for you. That's what computers are for, really. Remembering things.

What Views Do

  • Save your questions
  • Share them with others
  • Run them whenever you want
  • Get answers in different shapes

Making a View

Step 1: Go to a repository.
Step 2: Find the views section.
Step 3: Click New View.
Step 4: Type a name for your question.

Figure 1: Name your question.

Step 5: Press Enter.
Step 6: A little popup tells you it worked.

Figure 2: It worked. Now write the actual question.

Step 7: An editor appears. That's where you write your SPARQL.
Important: The view is empty right now. It doesn't do anything yet. You have to write something and save it. The computer is patient, but it can't read minds.

Writing in the Repository

You can edit views right there in the repository pane.

Opening the Editor

Step 1: Go to the repository.
Step 2: Click on the view name.
Step 3: An editor opens. Write your SPARQL there.

Writing the Query

Step 1: Find the text area.
Step 2: Type your SPARQL. The colors help you see what's what.

Figure 3: Write your question here.

Step 3: Click Save.
Step 4: It tells you it saved.
Tip: Ctrl+Enter runs the query. Cmd+Enter on a Mac. You don't have to save to run.

Writing in Its Own Space

You can give a view its own pane. More room to think.

Step 1: Go to the repository.
Step 2: Find the view.
Step 3: Click the pane button. It looks like a little folder.
Step 4: A new pane opens. More space.

Figure 4: Room to breathe.

Step 5: Write your SPARQL.
Step 6: Click Save.
Note: You can have lots of editors open. Some here, some there. They don't know about each other, and that's fine.

Running a View

You ask the question. The computer answers. That's the deal.

From the Editor

Step 1: Open the view in an editor.
Step 2: Make sure the query looks right.
Step 3: Pick an output format if you want.

Figure 5: Ready to ask.

Step 4: Click Run. Or press Ctrl+Enter.
Step 5: The answer appears below. There it is.

In a New Window

Sometimes you want to see results in their own window. Fresh page. Clean slate.

Step 1: Go to the repository.
Step 2: Find the view.
Step 3: Click the window button. It looks like a link.

Figure 6a: The window button.

Step 4: A new window opens with a table of results.

Figure 6b: Results in their own window.

Step 5: Pick different formats from the dropdown.
Step 6: Watch the results change.
Note: The new window needs to know who you are. The app handles that automatically.
Popups: If nothing happens, your browser might be blocking popups. Allow them for this site.

Deleting a View

Sometimes questions aren't worth asking anymore.

Warning: Gone is gone. Copy the query text somewhere if you might want it later.
Step 1: Find the view.
Step 2: Click the trash can.
Step 3: A confirmation appears.

Figure 7: Are you sure? Really sure?

Step 4: Read it.
Step 5: If you're sure, confirm.
Step 6: The view disappears from the list.

Editor Features

The editor does some helpful things.

Multiple Tabs

  • Each tab is a different query
  • Results stay with their tabs
  • Switch between them to compare
  • Close the ones you don't need

Reset

  • Click Reset to undo your changes
  • Goes back to the saved version
  • Everything you typed disappears

Event Log

  • Shows what happened
  • Queries you ran
  • Things you saved
  • Errors you caused
  • How long things took

Drag to Window

  • Grab an editor tab
  • Drag it outside the browser
  • Let go
  • New window with your query

Some Advice

  • Names: Give views names that mean something
  • Organization: Related views go in the same repository
  • Comments: Write notes in your SPARQL with #
  • Testing: Run queries before you save them
  • Backups: Copy important queries somewhere else

When Things Go Wrong

Won't Save

  • Did you click Save?
  • Check the event log
  • Do you have permission?
  • Is the internet working?

Won't Run

  • Check your SPARQL syntax
  • Are the URIs right?
  • Check the event log
  • Is there data in the repository?

View Missing

  • Refresh the page
  • Did it actually save?
  • Right repository?
  • Do you have read permission?
